# Hum-to-Beep（マイク音階入力）機能 要求仕様書

## 1. 目的
ユーザーがマイクに向かって鼻歌を歌ったり楽器を演奏したりすることで、そのメロディを自動的に解析し、ピアノロール上のノート（音符）として展開する機能を提供する。直感的なメロディ入力を可能にすることで、サウンドデザインの効率を大幅に向上させる。

## 2. 主要機能

### 2.1 マイク録音とリアルタイム解析
- **録音ボタン**: ピアノロールのコントロールエリアに「MIC RECORD」ボタンを追加する。
- **録音中のフィードバック**: 
    - 録音中は、現在の録音秒数をプログレスバーとして表示する。
    - プログレスバーの最大値は、現在編集中のパターンの設定秒数（例: 1s, 2sなど）とする。
- **自動停止**: パターンの設定秒数に達した時点で、録音と解析を自動的に終了する。

### 2.2 相対音階キャリブレーション（相対ピッチ）
- **最初の音をC4（基準音）に設定**: 
    - 録音された音声の**最初の有効なピッチ(周波数)**を、ピアノロール上の **C4**（内部周波数 4186Hz）として扱う。
    - 以降の音高の変化は、この最初の音を基準とした相対的な音程差として処理する。
    - これにより、ユーザーが本来のC4の高さで歌えなくても、自分の出しやすい音程でメロディを入力できる。

### 2.3 クオンタイズ（グリッド吸着）
- **時間軸のクオンタイズ**: 録音中のデータを解析し、本アプリの基本単位である **62.5ms (16分音符相当)** 単位でノートに分割する。
- **ピッチのクオンタイズ**: 解析された各区間の平均周波数を、最も近い**半音単位のスケール**（ピアノロールの行）に補正する。

### 2.4 ローカル処理
- 音声の取得からピッチ解析、ノートへの変換処理まで、すべての計算をブラウザ（クライアントサイド）内で完結させる。サーバーへの音声データの送信は行わない。

## 3. ユーザーインターフェース (UI)
- **コントロールバー**: 「MIC RECORD」ボタン（録音中は「STOP」に変化）を配置。
- **プログレス表示**: ピアノロール上部またはコントロールバー内に、録音の進捗を示す視覚的なバーを表示。
- **展開ボタン**: 録音終了後、解析結果を現在のパターンに上書き（または追加）して配置する。

## 4. 技術的な考慮事項
- **Web Audio API**: `AnalyserNode` を使用したピッチ検出アルゴリズム（自己相関法等）の実装。
- **ノイズ除去**: 一定以下の音量の無音区間は休符（ノートなし）として扱う。
- **倍速録音（Slow Recording）**: 
    - 録音時間をパターンの設定秒数の N 倍（例: 2倍、3倍）に引き延ばして録音できる機能を追加。
    - ユーザーはゆっくりと正確にメロディを歌い、解析時に元の速さに圧縮（クオンタイズ）してピアノロールに展開する。
- **録音カウントダウン**: 
    - 録音開始ボタンを押した後、3, 2, 1 のカウントダウンを表示してから実際の録音を開始する。
    - **先行録音**: カウントダウンの「1」から内部的に録音を開始。
    - **頭ぴったり録音 (Silence Trimming)**: 録音されたデータから最初の発音を検出し、その直前の無音区間を自動的にカットすることで、メロディがパターンの先頭から始まるように補正する。
- **ブラウザ許可**: ユーザーにマイクの使用許可を求めるプロンプトを適切に表示する。iOS/Android のブラウザでも動作するように考慮する。

## 5. モバイル・マルチデバイス対応
- **UIの最適化**: 録音ボタンやプログレスバーをモバイルの狭い画面でも押しやすく、見やすいサイズで配置する。
- **低負荷アルゴリズム**: スマートフォンのCPUでもリアルタイム解析ができるよう、軽量なピッチ検出アルゴリズムを採用する。
- **オーディオコンテキスト制約**: iOS Safari等の制約（ユーザー操作によるオーディオ再生開始が必要等）を考慮した実装を行う。
